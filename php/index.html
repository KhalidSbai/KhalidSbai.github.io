✦ Q1 — Classe abstraite Transportattributs + constructeur
💡 Rappel : Le mot-clé abstract empêche l'instanciation directe. On utilise protected pour que les sous-classes (Autocar) accèdent aux attributs.
✦ Q2 — Méthode infos()ajoutée dans Transport
💡 Rappel : Cette méthode sera redéfinie dans Autocar. On utilisera parent::infos() pour réutiliser ces lignes sans les réécrire.
✦ Q3 — Méthode abstraite montant()ajoutée dans Transport
💡 Rappel : Une méthode abstract n'a pas de corps {}. Elle force toutes les sous-classes à l'implémenter, sinon PHP génère une erreur fatale.
✦ Q4 — Classe Autocarhéritage + constructeur complet
💡 Rappel : extends = héritage. parent::__construct() appelle le constructeur de Transport pour initialiser idTrans, vitesse, capacite.
✦ Q5 — Redéfinir infos()dans Autocar avec parent::infos()
💡 Rappel : parent::infos() réutilise le code déjà écrit dans Transport. C'est le principe de réutilisabilité en OOP.
✦ Q6 — Implémenter montant()dans Autocar
💡 Formule : capacite × prixTicket → ex: 50 places × 150 DH = 7500 DH.
