B. Classe Logger — Pour développer des applications web efficaces, il est crucial de journaliser les actions des utilisateurs. Nous créerons une classe Logger dédiée à l'enregistrement des messages de journalisation d'un site web.
B.1.a — Constructeur de Logger 2 pts
Créez la classe Logger avec un constructeur qui prend en paramètre le nom du fichier de journalisation. Ce constructeur doit créer un flux de fichier pour la journalisation en utilisant le nom de fichier fourni, et il doit ajouter le texte "M107_EFM" au début du fichier.
B.1.b — Méthode log() 2 pts
Une méthode log($message) permet d'enregistrer un message dans le fichier de journalisation en incluant la date et l'heure actuelle sous le format : [YYYY-MM-DD HH:MM] $message.
B.2.a — Instance de Logger 1 pt
Créez une instance de la classe Logger en spécifiant un nom de fichier de journalisation (par exemple M107_EFM.txt).
B.2.b — Enregistrer 3 messages 2 pts
Utilisez la méthode log() pour enregistrer trois messages.
B.2.c — Lire et trier le fichier 2 pts
Lisez le contenu du fichier de journalisation M107_EFM.txt et affichez-le dans un tableau (objet array) trié en ordre décroissant des dates et heures des messages enregistrés pour vérification.
Partie pratique — 25 pts
L'OFPPT envisage une application web PHP pour simplifier la gestion des formations internes. Tables :
  • ResponsableFormation(matricule, motPasse, nom, prenom)
  • Formation(idFormation, titre, logo, dateDebut, DateFin, status, #matriculeResponsable)
Base de données : gestionFormations — serveur : localhost — port : 3306 — user : root — pwd : ''
A — Connexion à la base de données 2 pts
Écrivez un script PHP qui se connecte à la base de données « gestionFormations ». Assurez-vous que le script gère les erreurs de connexion.
B — login.php 3 pts
Écrivez un script PHP pour vérifier les données d'authentification. Redirigez l'utilisateur vers login.php avec un message d'erreur si le matricule ou le mot de passe est vide ou incorrect. Sinon, redirigez-le vers monCompte.php et créez une session avec la valeur du matricule. Les noms des champs d'entrée devraient être respectivement matricule et motpasse.
C.1 — Protection de monCompte.php 1 pt
Écrivez un script PHP qui gère l'accès à la page « monCompte.php », représentant une page à accès limité, où aucun visiteur n'a le droit de voir son contenu s'il n'a pas été authentifié par la page login.php.
C.2 — Affichage des formations 2+1+1 pts
Écrivez un script PHP qui complètera le code HTML pour afficher :
  • a. Une liste de formations, triées par date de début dans un ordre décroissant. (2pts)
  • b. La durée en jours calculée à partir de la date de début et de la date de fin. (1pt)
  • c. Le statut : "En cours" si la valeur est 0, "Terminée" si la valeur est 1. (1pt)
D — NouvelleFormation.php / insertionAction.php 2+1+1+1+2+1 pts
Écrivez un script PHP permettant de :
  • D1. Transférer le logo de formation vers le dossier « images ». (2pts)
  • D2a. Vérifier que l'identifiant respecte le format : F_2024_XXX (XXX = 3 chiffres). Afficher « Format invalide ! » sinon. (1pt)
  • D2b. Empêcher l'ajout si un enregistrement avec le même identifiant existe déjà → afficher « Déjà existant ». (1pt)
  • D2c. Vérifier que la date de début n'est pas future et que la date de fin est supérieure à la date de début → « Date invalide ». (1pt)
  • D2d. Insérer les données en associant la formation au responsable connecté (matricule). (2pts)
  • D3. Rediriger le responsable vers son espace privé. (1pt)
E — Annuler une formation 3 pts
Écrivez un script qui sera exécuté lorsque le responsable clique sur le lien « Annuler ». Ce script vérifie d'abord si la formation est déjà terminée. Si tel est le cas, afficher : « Impossible d'annuler une formation déjà terminée ! ». Sinon, la formation est supprimée de la base de données.
F — Terminer une formation 3 pts
Lorsqu'un responsable clique sur le lien « Terminer », la valeur de la colonne status est modifiée de 0 à 1. Si la formation est déjà terminée, un message indique que cette action a déjà été effectuée. Il est impossible de marquer une formation comme terminée si elle l'est déjà.
I — Déconnexion 1 pt
Écrivez un script qui, lorsqu'un responsable clique sur le lien « Se déconnecter », détruit la session en cours et redirige le responsable vers la page login.php.

💡 file() lit le fichier ligne par ligne dans un tableau. arsort() trie en ordre décroissant en préservant les clés. Les dates au format [YYYY-MM-DD HH:MM] se trient correctement par ordre alphabétique inversé.

💡 empty() détecte les champs vides. La requête préparée protège contre les injections SQL. $_SESSION['matricule'] stocke l'identifiant de l'utilisateur connecté.

💡 Regex : /^F_\d{4}_\d{3}$/ — ^ début, F_ littéral, \d{4} = 4 chiffres, _, \d{3} = 3 chiffres, $ fin.
✦ E — annuler.phpvérif statut + suppression
💡 On vérifie d'abord status == 1 (terminée) avant de supprimer. Si la formation est terminée → message d'erreur, sinon → DELETE + redirection.
✦ F — terminer.phpUPDATE status 0 → 1
💡 UPDATE ... SET status = 1 marque la formation comme terminée. On vérifie d'abord que status != 1 pour éviter une double action.
